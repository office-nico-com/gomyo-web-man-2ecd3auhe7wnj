|project| － |project_sub|
==========================================

概要
------------------------------------------
| 本書は、|project| の操作方法および運用の流れについて説明するものです。
| |project|  は、工事ごとの請負金額・原価・入金状況を一元管理し、損益を把握するための原価管理システムです。
| 各工事について、以下の情報を管理します。

- 請負金額
- 入金金額・未収金額
- 材料費・労務費・外注費などの原価
- 諸経費・管理費
- 損益・原価率

| これらを自動集計し、工事単位および発注者単位で収支状況を確認できます。
| また、工事台帳や発注者別集計表の帳票出力にも対応しており、月次管理や経営判断資料として活用できます。
| 本ツールにより、各工事の利益状況を可視化し、原価管理の精度向上を支援します。


利用の流れ
------------------------------------------
本ツールは、工事ごとの原価と損益を正確に把握するため、以下の流れで運用します。

#. 事前準備：正確な原価計算と入力の効率化を図るため、発注者・社員・品目などの基礎情報をマスタにあらかじめ登録します。
#. 工事登録：案件ごとの契約内容や基本情報を登録し、原価・請求・入金を紐づけて管理するための工事台帳を作成します。本ツールにおける管理の単位は「工事」です。
#. 原価入力：材料費、労務費、外注費、工事経費などを入力し、工事ごとの実行原価を把握します。これにより、進行中でも原価状況や利益見込みを確認できます。
#. 請求・入金：請求情報および入金情報を登録し、売上計上状況や未収金を管理し、資金回収状況を可視化します。
#. 集計・分析： 最後に、蓄積されたデータを集計し、収益性の分析や経営判断資料として活用します。工事別・発注者別の収益構造を把握することができます。
